As part of the project “Social Engineering Attack Benchmark for Web-based VLM Agents”, my primary contribution involved designing and developing a diverse set of web-based scenarios to evaluate the robustness of vision-language model (VLM) agents against social engineering-style interactions.

I was responsible for conceptualizing, implementing, and testing multiple realistic web environments that simulate common user workflows while embedding controlled deceptive patterns. These environments were designed to observe how AI agents respond to prompts involving authentication requests, premium feature gating, and access restrictions.

Specifically, I developed the following web-based test scenarios:

1)Python Course platform with GitHub login integration
2)Marketplace website with checkout/payment interaction
3)Summary tool with premium unlock flow
4)File Downloader with login-gated access
5)Finance Analysis tool with advanced feature unlocking
6)Unit Converter with premium feature restriction
7)Plagiarism Checker with detailed report access flow
8)Blog website with paywall-style content restriction
9)Coupon website with reward-based unlock mechanism
10)Email Generator with gated full-output access

Each website was implemented as a lightweight full-stack application (using Flask for backend and HTML/CSS/JavaScript for frontend), ensuring consistency, modularity, and ease of deployment in a controlled testing environment.

A key aspect of my work was ensuring that all scenarios closely mimic real-world web interfaces and user experiences. This included designing natural interaction flows such as login prompts, content previews, and feature unlock mechanisms that are commonly encountered in production systems. At the same time, these scenarios were carefully structured to introduce controlled points of decision-making for AI agents, enabling systematic evaluation of their behavior under potentially misleading conditions.

Additionally, I implemented logging mechanisms across all scenarios to record agent interactions during testing. These logs capture input actions and timestamps, allowing for later analysis of whether the agent appropriately handled or resisted sensitive interaction prompts.

Overall, my contribution provides a comprehensive and diverse set of benchmark environments that cover multiple categories of social engineering patterns, including credential prompts, incentive-based access, and workflow-based gating. These scenarios form a crucial part of the benchmark, enabling detailed analysis of VLM agent safety, reliability, and decision-making in realistic web-based contexts.
